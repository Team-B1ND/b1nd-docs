# Team-B1ND GitHub 규칙 통일 건의안

> 작성일: 2025-12-17
> 작성자: B1ND 운영팀
> 버전: 1.0

## 📌 건의 배경

Team-B1ND는 현재 30명의 개발자가 4개 팀(Android, Back-end, Front-end, iOS)으로 나뉘어 도담도담을 포함한 다수의 프로젝트를 운영하고 있습니다. 그러나 프로젝트별로 상이한 커밋 컨벤션, 이슈/PR 템플릿으로 인해 협업 효율성이 저하되고, 신규 팀원의 학습 곡선이 증가하는 문제가 발생하고 있습니다.

본 건의안은 **Team-B1ND 전체에 적용 가능한 통일된 GitHub 규칙**을 제안하여, 협업 품질 향상 및 개발 생산성 증대를 목표로 합니다.

## 🎯 건의 목적

1. **일관성 확보**: 모든 프로젝트에서 동일한 규칙 적용
2. **학습 곡선 감소**: 신규 팀원의 빠른 온보딩
3. **협업 효율 증대**: 명확한 커뮤니케이션 체계
4. **코드 품질 향상**: 체계적인 리뷰 프로세스

### [0. 현재 문제점 분석](./00-current-issues.md)
Team-B1ND의 현재 GitHub 운영 현황과 개선이 필요한 문제점을 데이터 기반으로 분석합니다.

### [2. 브랜치 전략](./02-branch-strategy.md)

### [3. 이슈 템플릿](./03-issue-templates.md)

## 🚀 빠른 시작

### 즉시 적용 가능한 항목

1. **커밋 컨벤션**: 다음 커밋부터 바로 적용
2. **브랜치 네이밍**: 새 브랜치 생성 시 규칙 준수
3. **PR 작성**: 템플릿을 복사하여 즉시 사용

### 설정이 필요한 항목

1. **이슈 템플릿**: `.github` 저장소에 YAML 파일 추가
2. **PR 템플릿**: 각 프로젝트 또는 `.github` 저장소에 템플릿 추가
3. **브랜치 보호**: GitHub 저장소 설정에서 규칙 적용
