Team-B1ND 이슈 규칙에 맞춰 GitHub 이슈를 생성해주세요.

## 이슈 제목 규칙

**작성 규칙**:
1. 한국어, 영어 모두 사용 가능
2. 명확하고 구체적으로 작성
3. **50자 이내**로 간결하게
4. 마침표 사용 금지
5. 라벨로 타입을 표시하더라도 제목에 type 포함 권장

**좋은 예시**:
```
웹뷰 브릿지 카메라 API 구현
로그인 시 토큰 갱신 오류 수정
알림 서비스 MSA로 분리
API 명세서 업데이트
```

**나쁜 예시**:
```
기능 추가 → 너무 모호함
카메라 기능을 추가하고 싶어요. → 마침표, 구어체
웹뷰 브릿지 카메라 API를 구현하고 네이티브 통신을 추가하며... → 너무 김
```

## 필수 라벨

### 1. Type 라벨 (필수)

| 라벨 | 설명 | 예시 |
|------|------|------|
| `type: feature` | 새로운 기능 추가 | 웹뷰 브릿지 카메라 API 구현 |
| `type: bug` | 버그/오류 | 로그인 시 토큰 갱신 오류 수정 |
| `type: refactor` | 코드 구조 개선 | 알림 서비스 MSA로 분리 |
| `type: docs` | 문서 추가/수정 | API 명세서 업데이트 |
| `type: chore` | 빌드, 설정 등 유지보수 | Docker 환경 설정 추가 |
| `type: enhancement` | 기존 기능 개선 | 검색 속도 개선 |
| `type: test` | 테스트 코드 | E2E 테스트 추가 |
| `type: hotfix` | 긴급 버그 수정 | 프로덕션 서버 다운 긴급 수정 |

### 2. Priority 라벨 (필수)

| 라벨 | 설명 |
|------|------|
| `priority: critical` | 긴급 - 서비스 중단, 즉시 처리 필요 |
| `priority: high` | 높음 - 중요한 기능, 우선 처리 |
| `priority: medium` | 보통 - 일반 작업, 정상 일정 |
| `priority: low` | 낮음 - 급하지 않음, 여유 시 처리 |

### 3. Size 라벨 (필수)

| 라벨 | 설명 |
|------|------|
| `size: XS` | 매우 작음 - 4시간 이내 |
| `size: S` | 작음 - 1-2일 |
| `size: M` | 보통 - 3-5일 |
| `size: L` | 큼 - 1-2주 |
| `size: XL` | 매우 큼 - 2주 이상 (분할 검토 필요) |

### 4. Init 라벨 (선택)

| 라벨 | 설명 |
|------|------|
| `init: project` | 프로젝트 초기화 |
| `init: structure` | 구조 초기화 |
| `init: docs` | 문서 초기화 |

## 이슈 본문 템플릿

```markdown
## 작업 내용
(무엇을 할 것인지 명확하게 설명합니다)

## 작업 목적
(왜 이 작업이 필요한지 설명합니다)

## 체크리스트
- [ ] 작업 항목 1
- [ ] 작업 항목 2
- [ ] 작업 항목 3

## 참고 자료
(관련 링크나 자료가 있다면 첨부합니다)
```

## 작업 순서

1. 제목 입력 받기 (인자로 제공되거나 대화형으로)
2. Type, Priority, Size 라벨 선택
3. 이슈 본문 작성 (~니다 체 사용)
4. `gh issue create` 명령으로 이슈 생성:
   ```bash
   gh issue create \
     --title "제목" \
     --body "본문" \
     --label "type: feature,priority: high,size: M"
   ```
5. 생성된 이슈 번호와 URL 반환

## 인자

제목: $ARGUMENTS

## 본문 작성 규칙

1. **~니다 체 사용** (~합니다, ~입니다)
2. 템플릿 주석 지우지 말 것
3. 템플릿의 모든 칸 작성
4. 명확하고 구체적으로 작성

## 금지사항

- ❌ 의미없는 제목 (ex: 테스트, 버그 수정)
- ❌ 본문 없이 제목만 작성
- ❌ 라벨 미지정
- ❌ 하나의 이슈에 여러 관련없는 작업 포함

이슈 생성 후 이슈 번호와 URL을 알려주세요.
